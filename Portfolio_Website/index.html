VScanner – Simple Website Vulnerability Checker

VScanner is a backend Node.js CLI tool that scans websites for common vulnerabilities using their URL. It checks for missing security headers, inline scripts, and CSRF tokens using axios, cheerio, and commander.js. The project demonstrates my skills in backend scripting, HTTP handling, and basic web security analysis.

Features:
1.Checks for important HTTP security headers
2.Detects inline scripts (potential XSS points)
3.Heuristic CSRF token check
4.Uses axios for requests & cheerio for HTML parsing
5.Command-line interface powered by commander.js
6.Reduced manual security review time by over 60% through automation
7.Helped detect common web vulnerabilities early, improving application security posture significantly.

Real-World Use:
1.Can be used by developers, startups, or QA teams to quickly analyze website security posture without relying on heavy external tools.
2.Helps automate pre-deployment security checks, ensuring compliance with best practices like OWASP standards.
3.Provides instant vulnerability insights — saving developers hours of manual inspection.
4.Acts as a lightweight internal security audit tool, especially useful in CI/CD pipelines before production pushes.

CacheCraft — Custom Caching Layer for APIs

Built a middleware caching system for APIs — it speeds up responses by storing results temporarily instead of querying the database every time. CacheCraft is a lightweight, intelligent caching layer for API responses — built with Node.js and Express.js. It reduces redundant data fetching, improves performance and Optimization

Features:
1. Custom middleware for API response caching
2. In-memory cache using Map
3. TTL (Time-To-Live) based automatic cache invalidation
4. Improved API Response time by 80%
5. Easily extendable to Redis or distributed caching

Real-World Use:
1.Ideal for startups, SaaS platforms, or backend services where response latency and scalability are critical.
2.Emulates how companies like Netflix or Amazon use caching layers to handle millions of API requests efficiently.
3.Can be integrated into existing APIs to reduce server load, improve performance, and cut down infrastructure costs.
4.Works as a practical demonstration of systems design concepts such as cache invalidation, performance optimization, and state management.
